## Topic

A design-oriented skill framework that tries to make coding agents better at visual judgment by packaging reusable design heuristics, commands, and context gathering.

## Research Summary

- Impeccable is no longer just a thread idea, it has a website, a GitHub repo, and independent write-ups describing its installation and command surface.
- The strongest claim is not that it makes models magically tasteful, but that it gives them a structured design vocabulary and repeatable workflows.
- That makes it a useful example of skill-based augmentation for non-code judgment tasks.

## Key Findings

- Design guidance becomes more durable when captured as reusable commands and supporting context instead of one-off prompts.
- Automatic context gathering is particularly valuable for design work because visual and layout mistakes often come from missing local context.
- The project reflects a broader shift from “AI can design” to “AI needs curated design scaffolding.”

## Practical Relevance (for Gonzalo)

- Useful if Gonzalo wants better design output from coding agents without turning every task into a manual art-direction session.
- Also a strong reference for how to package soft skills, not just technical ones, into reusable agent tooling.

## Risks / Caveats

- Design heuristics can become rigid or style-biased if treated as rules rather than guidance.
- The gains may be strongest on common web-app patterns and weaker on unusual visual systems.

## Recommended Next Steps

1. Borrow the packaging pattern even if not adopting Impeccable wholesale.
2. Test on one real interface task and compare with an unscaffolded agent baseline.
3. Keep human taste in the loop for final visual decisions.
